# KeyBuddy 세계관 설정 규칙

> 문서 상태: World Setting v0.2 / 2026-07-24
>
> 적용 범위: 캐릭터, 몬스터, 배경, 장비, 이펙트와 향후 콘텐츠 설정

이 문서는 KeyBuddy에 무엇이 등장할 수 있는지와, 서로 다른 세계의 콘텐츠를 어떤 공통 규칙으로
묶을지를 정의한다. 세부 시나리오를 미리 완성하는 문서가 아니라, 즉흥적으로 추가한 리소스가 서로
무관해지는 것을 막기 위한 **최소 세계관 및 제작 기준**이다.

캐릭터/몬스터의 픽셀 크기와 Unity 납품 규격은
[`character-sprite-and-animator-rules.md`](./character-sprite-and-animator-rules.md)를 우선한다.
세계관의 자유는 해당 그래픽·애니메이션 규격을 무시할 근거가 되지 않는다.

## 1. 한 문장 세계관

> 여러 세계에서 사용자의 데스크탑으로 넘어온 모험가들이 이세계 용병단을 결성하고, 데스크탑을
> 침식하는 버그와 버그에 감염된 이세계 몬스터를 처치하며 성장한다.

## 2. 세계관의 방향

- 사용자가 판타지 세계로 들어가거나 그 세계의 신으로 개입하는 구조가 아니다.
- 여러 세계의 존재가 **사용자가 있는 데스크탑 세계로 넘어오는 구조**다.
- 데스크탑은 모든 콘텐츠가 만나는 공통 허브다.
- 처음에는 판타지 계열 모험가를 중심으로 시작할 수 있지만, 특정 세계관 하나로 전체 확장을 제한하지 않는다.
- 새로운 연결 세계가 발견되면 현대, 동물, 외계 행성, 기계 문명 등 다른 종족·직업·장비·배경을 추가할 수 있다.
- 설정은 가볍고 이해하기 쉬워야 하며, 캐릭터 제작을 방해할 정도의 상세 연대기나 정치 설정을 먼저 만들지 않는다.

## 3. 핵심 용어

| 용어 | 의미 |
|---|---|
| 데스크탑 세계 | 사용자가 실제로 활동하는 컴퓨터 환경이자 게임의 공통 무대 |
| 연결 세계 | 차원 현상으로 데스크탑과 연결된 판타지·현대·동물·외계 등의 개별 세계 |
| 이세계 용병단 | 서로 다른 연결 세계에서 넘어와 데스크탑을 방어하는 캐릭터 집단 |
| 관리자 | 데스크탑의 소유자인 사용자. 용병단을 지원하고 성장 에너지를 제공하는 역할 |
| 디지털 변환 | 다른 세계의 존재가 데스크탑 환경에서 활동 가능한 작은 컴패니언 형태로 변환되는 현상 |
| 버그 | 데스크탑을 침식하는 적대적 오류 또는 오염 현상 |
| 감염체 | 연결 세계의 생물·사물·몬스터가 버그에 감염되어 적대화된 존재 |

고유 명칭은 추후 변경할 수 있다. 명칭보다 위 역할과 관계를 먼저 유지한다.

## 4. 사용자의 역할과 게임 플레이 연결

사용자는 이세계 인물의 삶을 직접 조종하는 절대자가 아니라, 데스크탑의 **관리자이자 지원자**다.

- 사용자의 작업과 키 입력에서 활동 에너지가 발생한다.
- 용병단은 이 에너지를 전투, 훈련과 성장에 사용한다.
- 사용자가 작업하는 동안 캐릭터는 화면 하단에서 대기하고 활동한다.
- 몬스터 출현은 데스크탑 내부의 버그 또는 연결 세계에서 넘어온 감염체의 침입으로 설명한다.
- 캐릭터 교체는 같은 거점에 소속된 용병단 구성원의 교대다.
- 장비와 의상 교체는 데스크탑에서 재구성된 용병 장비 또는 출신 세계의 장비 변형으로 설명할 수 있다.

시스템 구현이 먼저 바뀌었다는 이유만으로 세계관을 억지로 확장하지 않는다. 실제 게임 기능이 확정되면
이 항목과 연결되는 최소한의 설명만 추가한다.

## 5. 캐릭터 설정 규칙

### 5.1 허용 범위

연결 세계가 등록되어 있다면 다음을 포함한 다양한 캐릭터가 등장할 수 있다.

- 중세 판타지의 인간, 수인, 용인, 요정과 마법 생명체
- 동물 의인화 세계의 주민
- 현대 세계의 경찰, 요원, 기술자와 일반 직업군
- 미래 또는 외계 문명의 로봇, 사이보그와 외계 종족
- 동양 판타지, 공포, 스팀펑크 등 독립적인 테마 세계의 인물

단, “여러 세계가 연결된다”는 설정을 아무 디자인이나 무규칙하게 추가하는 이유로 사용하지 않는다.
모든 캐릭터는 출신 세계와 무관하게 KeyBuddy의 공통 아트·스케일·애니메이션 규격으로 디지털 변환된다.

### 5.2 캐릭터 추가 시 필수 최소 설정

새 캐릭터는 이미지 생성 전에 아래 정보를 기록한다.

```text
Actor ID / 표시 이름:
출신 세계 ID:
세계 유형: Fantasy | Animal | Modern | SciFi | Alien | 기타
종족:
직업 / 용병단 역할:
출신 세계를 설명하는 한 문장:
데스크탑에 넘어온 상태 또는 계기 한 문장:
원래 세계에서 유지된 시각 요소 3개:
디지털 변환 후 KeyBuddy 공통 규격에서 조정된 요소:
체형 등급:
실제 스케일:
```

캐릭터 한 명을 추가하기 위해 국가, 역사, 종교 전체를 먼저 만들 필요는 없다. 출신 세계를 구분할 수 있는
한두 문장과 시각 요소만 기록하고, 같은 세계 출신 캐릭터가 늘어날 때 공통 설정을 확장한다.

### 5.3 체형 비율과 실제 스케일 분리

- **체형 비율**은 머리와 몸의 비율 및 실루엣 유형을 의미한다.
- **실제 스케일**은 게임 세계에서 다른 Actor와 비교한 키와 체격을 의미한다.
- 스케일 2 캐릭터도 2~2.5등신 SD 체형일 수 있으며, 스케일 0.9 캐릭터도 상대적으로 늘씬한 체형일 수 있다.
- 종족만으로 인간은 3등신, 수인은 2.5등신처럼 절대 고정하지 않는다.
- 종족별 경향이 실제 리소스 비교를 통해 확인되면 별도의 아트 규칙으로 추가한다.

체형 등급의 구체적인 수치와 명칭은 여러 종족의 실제 스프라이트를 비교한 뒤 확정한다. 확정 전에는
Character Brief에 해당 Actor의 승인 비율을 개별 기록한다.

### 5.4 디지털 변환 공통 규칙

출신 세계가 달라도 다음 요소는 공통이어야 한다.

- KeyBuddy에서 승인된 SD 체형 범위와 픽셀 화풍
- 동일한 외곽선, 광원, 색면 밀도와 최종 픽셀 블록 규격
- 게임 내 상대 스케일 규칙
- 작은 데스크탑 표시 크기에서 구분 가능한 얼굴과 실루엣
- 애니메이션 생성에 적합하게 노출된 관절과 읽기 쉬운 장비 배치
- 과도한 신체 가림, 복잡한 겹침과 모션마다 위치가 전환되는 대형 장비 최소화

출신 세계의 화풍 자체를 그대로 가져오는 것이 아니라, **출신 세계의 디자인 특징을 KeyBuddy 시각 언어로
번역한다.**

## 6. 몬스터와 버그 설정 규칙

### 6.1 적의 두 종류

1. **버그 자체가 형체를 얻은 몬스터**
   - 데이터 조각, 오류 코드, 깨진 UI, 파일 잔해와 노이즈 등에서 태어난 존재
2. **버그에 감염된 이세계 존재**
   - 연결 세계의 생물, 몬스터, 식물, 기계 또는 사물이 데스크탑으로 넘어오며 감염된 존재

두 번째 유형을 적극적으로 사용하여 몬스터 종류가 디지털 벌레나 슬라임에만 제한되지 않게 한다.
허수아비, 늑대인간, 골렘, 유령, 기계 생명체 등 어떤 원형도 감염체가 될 수 있다.

### 6.2 공통 감염 표식

서로 다른 몬스터가 같은 적대 세력으로 읽히도록 아래 표식 중 필요한 것만 1~2개 사용한다.

- 일부 색상이 비정상적으로 치환되거나 깨진 영역
- 신체 또는 장비에 생긴 작은 디지털 균열
- 픽셀 노이즈로 흩어지는 가장자리
- 오류 색상으로 발광하는 눈, 상처 또는 핵
- 피격·등장·사망 시 짧게 나타나는 글리치 연출

모든 몬스터를 항상 심한 글리치로 덮지 않는다. 원래 종족과 직업 실루엣이 먼저 읽혀야 하며,
버그 표현은 공통 소속을 알려주는 보조 표식이다.

### 6.3 몬스터 추가 시 최소 설정

```text
Monster ID / 표시 이름:
원래 출신 세계 ID 또는 Desktop Native:
원래 존재 유형:
감염 전 모습 한 문장:
버그 감염 표식:
데스크탑에서의 전투 역할:
실제 스케일:
```

## 7. 배경과 무대 규칙

- 데스크탑은 모든 콘텐츠가 돌아오는 기본 허브다.
- 평상시 캐릭터는 실제 데스크탑 또는 데스크탑을 단순화한 무대에서 활동한다.
- 전투 중 연결 세계의 지형 조각, 차원문 또는 데이터화된 배경이 데스크탑 위에 투영될 수 있다.
- 판타지 숲, 현대 도시, 우주 시설 등 서로 다른 배경 테마를 사용할 수 있다.
- 배경이 달라져도 데스크탑 UI, 차원 연결 연출, 버그 오염 표현 중 하나 이상으로 공통 세계와의 연결을 보여준다.
- 특정 세계의 배경 화풍을 추가할 때에도 캐릭터 가독성과 화면 하단 컴패니언 기능을 방해하지 않는다.

## 8. 연결 세계 등록 방식

새로운 출신 세계는 해당 세계의 첫 캐릭터 또는 몬스터가 실제 제작 단계에 들어갈 때만 등록한다.
아이디어만 있는 세계를 미리 대량으로 만들지 않는다.

```text
World ID:
임시/정식 이름:
유형:
한 문장 설명:
주요 종족 또는 존재:
기술·마법 수준:
공통 복식·재질·색상 키워드:
데스크탑과 연결된 방식:
소속 Actor:
상태: Concept | Active | Hold
```

### 8.1 등록된 연결 세계 — 애니멀랜드

```text
World ID: ANIMAL-LAND-01
정식 이름: 애니멀랜드 (Animal Land)
유형: Animal / Medieval Fantasy
한 문장 설명: 의인화된 동물 종족들이 살아가며, 종족 단위로 선과 악의 진영이 명확하게 나뉜 중세 판타지 세계.
주요 존재: 의인화 동물 종족, 용인족과 판타지 생명체
기술·마법 수준: 중세 판타지 / 검, 활, 갑옷, 마법과 성직 체계 사용
데스크탑과 연결된 방식: 상세 설정 보류
상태: Active
```

#### 세계 규칙

- 애니멀랜드의 주요 문명 구성원은 의인화된 동물 종족이다.
- 선과 악의 진영 구분이 비교적 명확한 세계다.
- 진영은 개인 직업보다 **동물 종족 단위의 소속**을 기본으로 양분된다.
- 어느 종족이 악의 진영에 속하는지는 해당 종족 캐릭터나 몬스터를 실제 제작할 때 추가한다.
- 현재 등록된 플레이어 캐릭터는 모두 선의 진영에 속한다.
- 중세 판타지 복식과 장비를 기본으로 하되, 동물의 귀, 주둥이, 꼬리, 날개 등 종족 실루엣을 유지한다.
- 종족 특징이 장비에 완전히 가려지지 않게 하며, 애니메이션에 필요한 손과 관절의 가독성을 우선한다.

#### 현재 소속 캐릭터

| Actor | 종족 / 역할 | 진영 | 리소스 상태 |
|---|---|---|---|
| DogShieldWarrior | 강아지 수인 / 방패전사 | 선 | Hold |
| BlackCatMage | 검은 고양이 수인 / 마법사 | 선 | Active |
| CatKnight | 고양이 수인 / 기사 | 선 | Existing |
| DragonWarrior | 용인족 / 양손검 전사 | 선 | Concept |
| FoxArcher | 여우 수인 / 궁수 | 선 | Concept |

리소스가 보류되거나 교체되더라도 위 캐릭터의 세계관 소속은 별도 변경 전까지 유지한다.

### 8.2 등록된 연결 세계 — 인간 판타지 세계(명칭 미정)

```text
World ID: HUMAN-FANTASY-01
임시 이름: 인간 판타지 세계 (정식 명칭 미정)
유형: Human / Medieval Fantasy
한 문장 설명: 인간 왕국들이 영토와 이해관계를 두고 빈번하게 전쟁하며, 다양한 직업의 인간 용병들이 활동하는 세계.
주요 존재: 인간
기술·마법 수준: 일반적인 중세 판타지 / 무기, 갑옷, 마법, 성직과 왕국 체계 사용
데스크탑과 연결된 방식: 상세 설정 보류
상태: Active / Name TBD
```

#### 세계 규칙

- 현재 기준 주요 지성 종족은 인간이며, 종족 진영 개념을 사용하지 않는다.
- 주요 갈등은 인간 왕국 간의 영토, 권력과 이해관계에서 발생한다.
- 왕국 간 전쟁이 빈번하여 정규군 외에도 다양한 직업의 용병이 활동한다.
- 전사, 마법사, 사제, 법관, 기사, 도적과 기타 인간 판타지 직업을 폭넓게 추가할 수 있다.
- 각 왕국의 이름, 정치, 종교와 전쟁사는 실제 캐릭터 배경에 필요할 때만 최소 단위로 추가한다.
- 정식 세계 명칭은 추후 해당 세계 캐릭터들의 공통 배경을 구체화할 때 확정한다.

#### 현재 소속 캐릭터

| Actor | 종족 / 역할 | 소속 세력 | 리소스 상태 |
|---|---|---|---|
| CopperAxeBarbarian | 인간 / 쌍도끼 바바리안 | 왕국·용병단 미정 | Active |
| FemaleMagistratePriest | 인간 / 법관·사제 | 왕국·교단 미정 | Concept |

### 8.3 아직 등록되지 않은 확장 세계 유형

아래 항목은 확장 가능성만 열어둔다. 실제 캐릭터 또는 몬스터가 제작 단계에 들어가기 전까지 개별
세계 설정으로 확정하지 않는다.

| 유형 | 사용 예 | 상태 |
|---|---|---|
| Modern | 경찰, 요원, 현대 직업군 | 확장 후보 |
| SciFi / Alien | 로봇, 외계 종족, 우주 문명 | 확장 후보 |
| 기타 테마 | 동양 판타지, 공포, 스팀펑크 등 | 확장 후보 |

## 9. 콘텐츠 제작 전 세계관 체크리스트

캐릭터·몬스터·배경 콘셉트를 승인하기 전에 확인한다.

1. 출신 세계 또는 Desktop Native 여부가 기록됐는가
2. 이 존재가 데스크탑에 있는 이유를 한 문장으로 설명할 수 있는가
3. 용병단 구성원인지 버그/감염체인지 역할이 분명한가
4. 출신 세계의 시각 특징이 3개 이내의 명확한 요소로 정리됐는가
5. 서로 다른 원작 화풍이 아니라 KeyBuddy 공통 시각 언어로 변환됐는가
6. 체형 비율과 실제 스케일을 별도로 기록했는가
7. 몬스터라면 버그 감염 표식이 과하지 않게 정의됐는가
8. 설정이 현재 게임 기능보다 지나치게 앞서거나 제작을 제한하지 않는가

## 10. 설정 변경 원칙

- 핵심 전제인 `데스크탑 허브`, `이세계 용병단`, `버그/감염체`는 프로젝트의 상위 설정으로 유지한다.
- 연결 세계, 종족, 직업과 지역 설정은 콘텐츠 추가 시 점진적으로 확장한다.
- 새 설정은 기존 설정을 무효화하기보다 빈 영역을 채우는 방식으로 추가한다.
- 실제 게임에서 사용하지 않을 상세 설정은 보류한다.
- 상위 전제를 변경해야 한다면 이 문서의 버전을 올리고 영향받는 캐릭터·몬스터·배경을 함께 기록한다.

## 11. 현재 제작 기준 요약

```text
Hub: 사용자의 데스크탑 세계
Player Role: 데스크탑 관리자 / 용병단 지원자
Allies: 여러 세계에서 디지털화되어 넘어온 이세계 용병단
Enemies: 버그 자체 또는 버그에 감염된 이세계 존재
Growth Source: 사용자의 작업과 키 입력에서 발생하는 활동 에너지
Expansion: 새 연결 세계 단위로 종족·직업·몬스터·배경 테마 추가
Art Rule: 설정은 자유롭게, 최종 표현은 KeyBuddy 공통 시각 언어로 통일
```
