# KeyBuddy (MVP)

가칭: KeyBuddy. (저장소/폴더명은 desktop_RPG로 남아있지만 프로젝트 명칭은 KeyBuddy를 사용한다.)

이 문서는 이전 "수련용사 키우기" 게임 기획안을 대체한다. 기존 문서(README, `Docs/tech-spike-transparent-window.md` 기획 서술)는 폐기했다.

## 프로젝트 목표

이 프로젝트는 완성형 게임을 만드는 것이 아니다.

최우선 목표는 사용자가 PC에서 작업하는 동안 화면 한쪽에서 작은 캐릭터가 함께 성장하는 즐거움을 제공하는 것이다.

현재 단계에서는 재미 검증이 목적이며, 모든 기능은 MVP 기준으로 최소화한다.

추후 확장성은 고려하되 현재는 구현하지 않는다.

## 프로젝트 철학

이 프로젝트는 "방치형 RPG"가 아니다.

"작업하는 시간을 조금 더 즐겁게 만드는 데스크탑 컴패니언"이다.

모든 기능은 이 목표를 해치지 않는 선에서만 추가한다.

기능이 늘어나는 것보다 사용자가 캐릭터를 한 번 더 쳐다보게 만드는 것이 더 중요하다.

재미는 복잡함이 아니라 작은 반응에서 나온다.

항상 MVP 기준으로 사고한다.

## 핵심 컨셉

- 데스크탑 화면 위에서 항상 동작하는 작은 캐릭터
- 사용자의 키보드 입력을 감지하여 공격
- 공격을 통해 적을 처치
- 경험치를 획득
- 레벨업

이 5개의 루프만 완성한다.

## 현재 구현 범위

- 캐릭터 1종
- 적 1종
- 공격 애니메이션
- 피격 애니메이션
- 사망 애니메이션
- 경험치
- 레벨업
- HP
- 데미지
- 콤보

끝.

## 게임 루프

```
사용자가 키보드를 입력한다.
        ↓
캐릭터가 공격한다.
        ↓
적이 데미지를 받는다.
        ↓
적이 죽으면 즉시 새로운 적 생성
        ↓
경험치 획득
        ↓
레벨업
        ↓
공격력 증가
        ↓
반복
```

## 적

현재는 허수아비 하나만 존재.

- 움직이지 않는다.
- 공격하지 않는다.
- 반격도 없다.
- 단순한 공격 대상이다.

## 캐릭터

- 캐릭터는 자동 이동하지 않는다.
- 허수아비 앞에서 계속 공격한다.
- 입력이 없으면 Idle.
- 입력이 발생하면 Attack.

## 키 입력

모든 키 입력을 동일하게 처리한다.

- 문자
- 숫자
- 백스페이스
- 엔터

모두 동일. 입력 내용은 중요하지 않다. 키 입력 횟수만 사용한다.

## 콤보

- 연속 입력 시 콤보 증가.
- 일정 시간 입력이 없으면 콤보 초기화.
- 콤보가 높을수록 공격 이펙트 / 화면 흔들림 / 데미지 숫자 등의 연출만 강화한다.
- 실제 밸런스에는 최소한만 영향.

## UI

최소 UI.

- LV
- EXP
- HP
- Combo

이 정도만 표시. 디버그 수준이면 충분.

## 리소스

최종 리소스가 아니다. AI 생성 리소스 또는 임시 리소스를 적극 사용한다. 현재 목적은 재미 검증이다. 리소스 퀄리티보다 교체 가능한 구조가 더 중요하다.

## 코드 원칙

항상 작은 단위로 구현.

```
기능 하나 구현 → 동작 확인 → 다음 기능 구현
```

한 번에 여러 시스템을 만들지 않는다.

## 가장 중요한 목표

30분 이상 타이핑하면서 캐릭터를 계속 보고 싶다는 느낌이 드는지 검증하는 것이 이번 MVP의 성공 기준이다.

게임의 분량이 아니라 캐릭터의 존재감과 타이핑의 손맛을 검증한다.

## 기존 구현물 (참고)

`Assets/Scripts/DesktopWindow`에 데스크탑 오버레이용 투명/테두리 없는 창 처리 코드(Windows 전용)가 이미 있다. "화면 한쪽에서 항상 동작하는 캐릭터"라는 핵심 컨셉과 직접 관련된 기반 코드이므로 유지하되, 이번 MVP 범위(위 5단계 루프) 밖의 신규 기획 요소는 추가하지 않는다.
